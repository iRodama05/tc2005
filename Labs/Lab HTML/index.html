Respuestas del Cuestionario

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

El Internet es la red física global de computadoras interconectadas. La World Wide Web (WWW) es un servicio que funciona sobre Internet para acceder a documentos mediante el protocolo HTTP.

2. ¿Cuáles son las partes de un URL?

Protocolo (ej. https://), Subdominio (ej. www.), Dominio (ej. sitio.com), Puerto (ej. :80), Ruta (/pagina.html), Parámetros (?id=1) y Fragmento (#seccion).

3. ¿Cuál es el propósito de los métodos HTTP?

  • GET: Obtener información.
  • HEAD: Obtener solo los encabezados, sin el cuerpo.
  • POST: Enviar información nueva (como un formulario).
  • PUT: Reemplazar por completo un recurso.
  • PATCH: Modificar parcialmente un recurso.
  • DELETE: Eliminar un recurso.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML? ¿Por qué?

Se debe utilizar POST. Porque envía los datos (como una contraseña) de forma oculta en el cuerpo del mensaje, y no expuestos en la URL como lo hace GET.

5. ¿Qué método HTTP se utiliza cuando se accede a una página a través de un URL?

Se utiliza el método GET.

6. Un servidor devuelve código 200. ¿Qué significa? ¿Ocurrió algún error?

Significa "OK". La solicitud fue exitosa y no ocurrió ningún error.

7. ¿Es responsabilidad del desarrollador corregir un error 404?

El error 404 significa "No encontrado". Es responsabilidad del desarrollador solo si el error ocurre porque hay un enlace interno roto en su propio sitio. Si el usuario escribió mal la URL, no es su responsabilidad.

8. ¿Es responsabilidad del desarrollador corregir un error 500?

Sí. Un error 500 (Internal Server Error) significa que el servidor o el código de la aplicación fallaron. Es una falla del sistema que debe repararse.

9. ¿Qué significa que un atributo esté desaprobado (deprecated)? Ejemplos:

Significa que la tecnología es obsoleta y ya no se recomienda su uso en HTML5. Ejemplos de HTML4 desaprobados son las etiquetas center, font, b (reemplazada por strong) o atributos como bgcolor.

10. ¿Qué componentes de estructura y estilo tiene una tabla?

Estructura: table, thead, tbody, tr (filas), th (encabezados) y td (celdas).
Estilo: En HTML5 puro el estilo no debe ir en las etiquetas; se debe utilizar CSS externo.

11. ¿Cuáles son los principales controles de una forma HTML5?

La etiqueta input (con tipos como text, number, email, radio, checkbox, submit), textarea, y select.

12. ¿Cuál es el ciclo de vida y desarrollo de sistemas de información?

El ciclo de vida es todo el tiempo que existe el sistema, desde que surge la necesidad hasta que es retirado. El ciclo de desarrollo son las fases específicas para crearlo: Planificación, Análisis, Diseño, Implementación, Pruebas y Despliegue.
