¿Qué ventajas ofrecen las variables CSS sobre los valores hardcodeados?
Facilitan enormemente el mantenimiento del código, ya que te permiten definir un valor (como un color o un margen) en un solo lugar y reutilizarlo en todo el proyecto. Si necesitas cambiar ese color en el futuro, solo modificas la variable y el cambio se aplicará automáticamente en todos los elementos que la usen.

¿En qué casos conviene declarar variables CSS fuera del selector :root?
Es útil cuando quieres limitar el alcance lo que uede hacer esa variable a un componente específico. Por ejemplo, si tienes una tarjeta y quieres que ciertas variables de color o espaciado solo existan y afecten a los elementos dentro de esa tarjeta, las declaras en la clase de ese contenedor en lugar de hacerlo de forma global.

¿Cómo se pueden utilizar las variables CSS para implementar un sistema de temas (como modo oscuro)?
Defines tus variables globales en el :root con los colores de tu tema claro. Luego, creas una clase específica (como .dark) que se aplica al body o al html, y dentro de esa clase vuelves a declarar las mismas variables, pero con los colores correspondientes al modo oscuro. Al alternar esta clase con JavaScript, toda la página cambia de tema instantáneamente.

¿Qué sucede cuando intentas usar una variable CSS que no ha sido declarada?
El navegador simplemente ignora esa declaración y el elemento tomará el valor heredado o el predeterminado.

¿Cuál es la diferencia entre los enfoques mobile-first y desktop-first?
Mobile-first consiste en escribir los estilos predeterminados para pantallas pequeñas (móviles) y usar media queries con min-width para ir añadiendo complejidad conforme la pantalla crece. Desktop-first hace exactamente lo opuesto: se diseña la versión completa para pantallas grandes y se usan media queries con max-width para ir simplificando el diseño en pantallas más pequeñas.

¿Qué breakpoints son los más comunes y por qué?
Los más estandarizados suelen ser alrededor de los 768px para identificar el paso a tablets, y 1024px o 1280px para pantallas de escritorio.

¿Cómo afecta el viewport meta tag al diseño responsivo?
Obliga a que el ancho de la ventana gráfica coincida con el ancho físico del dispositivo, permitiendo que tus media queries funcionen correctamente.

¿Qué técnicas además de media queries existen para crear diseños responsivos?
Puedes apoyarte mucho en el uso de unidades relativas como porcentajes, vw (ancho de ventana) o vh (alto de ventana). Además, herramientas como Flexbox y CSS Grid, combinadas con funciones matemáticas como clamp(), min() y max(), permiten crear elementos que se encogen y estiran sin necesidad de establecer puntos de quiebre rígidos.

¿Cómo puedes identificar qué regla CSS está siendo aplicada cuando hay múltiples selectores?
Los identificadores (IDs) pesan más que las clases, y las clases pesan más que las etiquetas HTML. Si dos selectores tienen exactamente el mismo peso, se aplicará el que haya sido escrito más abajo en el código.

¿Qué información proporciona la pestaña Computed en DevTools?
Muestra los valores finales que el navegador realmente aplica a un elemento tras calcular todas las reglas CSS. También incluye un esquema visual con las medidas exactas de sus márgenes, bordes y rellenos.

¿Cómo puedes simular diferentes dispositivos sin tener acceso físico a ellos?
Todos los navegadores modernos incluyen una herramienta dentro de sus DevTools que te permite visualizar tu página simulando las dimensiones de distintos teléfonos y tablets. Además, emulan características físicas como eventos táctiles en lugar de clics de ratón.

¿En qué situaciones es preferible usar Bootstrap sobre Tailwind CSS y viceversa?
Bootstrap es ideal si necesitas levantar un prototipo rápido o un panel de administración porque te da componentes completos ya diseñados (botones, modales, barras de navegación). Tailwind CSS es la mejor opción cuando tienes un diseño a medida y quieres tener control absoluto sobre cada detalle visual sin tener que luchar para sobrescribir los estilos que Bootstrap trae por defecto.

¿Qué significa que Tailwind sea "utility-first"?
Significa que en lugar de escribir tus propias clases CSS semánticas (como .tarjeta-perfil), construyes el diseño aplicando múltiples clases pequeñas directamente en tu HTML, donde cada clase tiene un único propósito visual (como dar color, añadir margen o cambiar el tamaño de la fuente).

¿Cómo afecta el uso de un framework CSS al tamaño final del bundle?
Un framework tradicional hace que el archivo sea muy pesado porque incluye código que no usas. Frameworks modernos (como Tailwind) solucionan esto eliminando automáticamente lo que no necesitas, dejando un archivo final muy ligero.

¿Es posible combinar múltiples frameworks CSS en un mismo proyecto? ¿Por qué sí o por qué no?
Técnicamente es posible, pero no es recomendable. Hacerlo aumenta enormemente el peso del sitio, causa conflictos impredecibles cuando ambos frameworks intentan modificar los mismos elementos globales, y hace que el código sea inconsistente y difícil de mantener para cualquier otro desarrollador.
